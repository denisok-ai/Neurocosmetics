# Page Layout Designer

## Принципы выравнивания Hero-секций

### Двухколоночный layout (текст + изображение)

1. **Верхнее выравнивание** — оба блока должны начинаться на одной вертикальной линии.
2. **Предпочтительно Flexbox** для предсказуемого `align-items: start`:
   ```html
   <div className="flex flex-col gap-16 lg:flex-row lg:items-start lg:gap-16">
     <div>Текст</div>
     <div className="flex shrink-0 items-start">Картинка</div>
   </div>
   ```
3. **Избегать** отрицательных margin — они хрупки и зависят от контекста.
4. **Grid** — при использовании `grid` обязательно `items-start` и `content-start` для выравнивания по верху.

### Чеклист при жалобах на «съехало»

- [ ] Оба столбца имеют `items-start` / `align-self: start`
- [ ] Нет лишнего `padding-top` или `margin-top` у правой колонки
- [ ] Контейнер использует `flex` или `grid` с явным `items-start`
- [ ] На мобильных — вертикальный стек (`flex-col`), на десктопе — ряд (`lg:flex-row`)

### Типичные ошибки

- `items-center` в двухколоночном hero — центрирует по вертикали, блоки «плывут»
- `justify-center` на правой колонке без `items-start` — картинка может центрироваться по обеим осям
- Разный `padding` у левой и правой колонок
